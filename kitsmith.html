KitSmith

A local Windows desktop app that turns audio you own into a vocal-free producer sound kit — stems, one-shots, MIDI, and analysis — fully offline. Ships as native MSI/NSIS installers.

The problem

Producers want stems, loops, and MIDI out of reference audio they have the rights to — but cloud tools mean uploading copyrighted audio, and they don't run offline.

How it works — three processes, one app

  1. A Tauri + React desktop UI the user interacts with.
  2. A thin Node HTTP bridge — the key decision for Windows reliability — relays to…
  3. A Python DSP worker (librosa/scipy/numpy) that separates stems, analyzes BPM/key/sections, slices one-shots, and generates MIDI, then exports a ZIP kit. Job state lives in SQLite; high-quality ML separation is a user-approved provider slot.

The story · STAR

S

Situation

As a producer I wanted stems, loops, and MIDI out of reference audio I own, but cloud tools mean uploading copyrighted audio, and I wanted everything offline.

T

Task

Build a local Windows desktop app that turns authorized audio into a usable sound kit, end to end.

A

Action

Built the UI in Tauri + React, but Python did the DSP — and getting them to talk reliably on Windows was the hard part, so I put a small Node HTTP bridge between them. The worker separates stems, analyzes, slices, and generates MIDI, then exports a ZIP. I kept job state in SQLite, abstracted separation behind a provider interface, added a Playwright smoke test, and packaged it into native MSI/NSIS installers.

R

Result

A feature-complete app with working Windows installers — the full upload-to-export pipeline runs locally and was verified on 2026-06-19. The lesson was architectural: when one runtime fights the platform, a thin bridge process can be cleaner than forcing it.
